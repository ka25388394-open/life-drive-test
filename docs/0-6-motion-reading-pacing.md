# 0-6｜Motion & Reading Pacing Specification

# 《你靠什麼感覺自己正在活著？》

## V1 Motion & Reading Pacing Specification　（rev.2 — 納入 Owner 優化建議）

> 本文件把 0-5 的**情緒體驗**,轉譯成:出現順序、停頓、閱讀節奏、微動畫、動畫 Token、Result Sequence。
>
> 本文件定義的是**動效意圖與節奏**,不是 CSS 實作。所有時間值延續並不得牴觸 0-4 §10 Motion。

---

# 0. Principle｜核心原則

- Animation **不是裝飾**。
- Animation 的唯一目的:**幫助使用者慢慢理解自己。**
- 判準:**Emotion > Efficiency。**
- 允許:**停頓、等待、留白。**

**Motion Priority｜動畫優先順序**

```
1. Reading（閱讀最優先）
2. Emotion（其次是情緒）
3. Visual（最後才是視覺）
```

任何動畫若影響閱讀,一律**降低或移除**。動畫的目的是**讓人慢慢閱讀**,不是讓畫面更酷。

**允許停留原則｜Allow the Pause**

> 若某段內容可能讓使用者產生:
>
> 「原來如此。」
> 「好像真的是這樣。」
> 「我最近好像就是這樣。」
>
> 則**允許停留**。不要為了流程流暢,而立即進入下一段。

整體感受目標:

> 像深夜,安靜地翻完一本屬於自己的小書;
> 最後,有一點溫暖地離開。
>
> 而不是:快速讀完一份心理測驗報告。

情緒與視覺基準:

| 來源 | 承接 |
| --- | --- |
| 0-4 §10 Motion | 250–350ms、fade + translateY(10px)、不滑入、不彈跳、不花俏 |
| 0-4 §9 Result | 慢慢淡入,不要一次全部出現 |
| 0-5 §2 / §7 | 五階段情緒歷程;像深夜翻完一本自己的小書 |

禁止(承 Owner 指示 + 0-4):

- ❌ 炫技動畫
- ❌ 遊戲感動畫
- ❌ 高刺激動畫
- ❌ 滑入 / 彈跳 / 花俏 / 閃爍 / 彩帶 / 視差 / 捲動劫持 / 載入型旋轉動畫

---

# 1. Homepage Sequence｜首頁出現順序

> 註:首頁(0-4 §7)尚未實作,本節先定義其動效意圖。

進站時先是安靜的留白,再**逐一**淡入,不同時出現。加入**品牌停頓**:

```
留白
 → 主標
 → 停頓
 → 副標
 → 停頓
 → 🌱
 → 停頓
 → 開始測驗
```

- 🌱 已逐漸成為產品的重要情緒符號,在此獨立出現、獨立停頓,讓它被看見。
- 情緒目標:像被溫柔地邀請,而不是被催促開始。「開始測驗」最後出現,讓「開始」成為使用者自己準備好後的選擇。

---

# 2. Question Transition｜題目切換

- 每次只呈現一題;前一題**先淡出**,一個短暫停頓後,下一題**淡入**(fade + translateY 10)。
- **不使用水平滑動**(承 0-4 §10)。切換是「翻頁」的安靜感,不是「滑走」的效率感。
- 「第 n / 8 題」與題目一起安定出現;選項可略晚一拍逐一淡入,讓視線自然落在題目上。
- 一題必須**完全安定**後,使用者才在自己的節奏下作答;動效不搶在閱讀之前。
- 提交後不立即彈出下一題,允許一個短停頓,像翻頁的手勢。

---

# 3. Result Reveal Sequence｜結果揭曉順序

- 按下「看結果」後,先有一個**較長的停頓**(像屏息的一拍),再開始揭曉——不立即把整頁丟出。
- 各區塊**逐一淡入,不一次全部出現**(承 0-4 §9),每塊之間留一個呼吸的停頓。
- 揭曉順序(依 0-4 §9 的情緒順序):
  1. 🌱 Icon
  2. 類型名稱
  3. 百分比
  4. 雷達圖(可略慢,讓比例「長出來」而非彈出)
  5. 文案
  6. 結尾
- 情緒目標:像一頁一頁翻開,而不是被交付一張報告。
- **待裁決:** 0-4 §9 的區塊順序與 0-2 §11 定稿結構不同(先前已標記)。本節描述的是「逐一、慢慢揭曉」的節奏;實際區塊順序以 Owner 對該衝突的最終裁定為準。

---

# 4. Reading Motion｜閱讀時的動態

- 內容**一旦出現就安定不動**:沒有捲動視差、沒有隨捲動觸發的位移、沒有持續晃動的元素。
- 閱讀階段的動態趨近於**零**——安靜,是為了讓使用者專心理解自己。
- 動效只發生在「出現的那一刻」,之後就讓文字靜靜待著,像書頁不會自己動。

---

# 5. Important Sentence Motion｜重要句子的抵達

- 重點句(如 0-3 中的引言句、0-5 的核心句)**晚一拍**、以**略慢**的淡入抵達,讓它「落下來」被看見。
- 效果仍是同一種 fade + translateY,**不換特效**——不放大、不高亮閃動、不變色跳動。
- 手法是**時間的留白**,不是視覺的強調。給一句話多一點停頓,就是給它重量。

---

# 6. Reading Pause｜情緒高點的停留

- 有些句子本身就是**情緒高點**,它們**不需要任何額外動畫**。
- 它們需要的是:**單純停留 1~2 秒**,不急著進入下一段。
- 讓使用者有時間產生:

  > 「啊……」
  > 「原來如此。」

- 情緒高點示意(引自現有文案精神,**不新增、不改寫**,僅標示這類句子該被停留):
  - 「你容易透過理解世界,感受到自己正在活著。」(0-3 Exploration)
  - 0-3 §4 結尾——關於「已經有一段時間,沒有好好靠近那些讓自己感受到生命力的事物」的段落。
- 作法:此類句子出現後,**延後下一段的淡入**,讓它獨自安靜地停一會兒(對應 Motion Token 的 `hold.insight`)。
- 與 **Motion Priority** 一致:此刻 Reading 最優先,寧可等待,也不要為了流程流暢而立刻推進。

---

# 7. Ending Sequence｜結尾節奏

- 固定結尾文案(0-3 §4)是**最後、最慢**出現的一段,前面留最長的停頓。
- 🌱 那一句以最輕柔的節奏抵達,然後——**停下來**,不再有任何動態。
- 「重新測驗」按鈕在結尾**再晚一段**才淡入,避免讓人感覺被催著離開;讓使用者有時間與那段話獨處。
- 情緒目標:讀完後,願意安靜地坐一會兒,再溫暖地離開。

---

# 8. Motion Token｜動畫語彙(語意 Token)

> 語意層 Token,延續 0-4 §10;供實作時對應,不在此寫 CSS。

**Duration(時長)**

```
enter.fast    ≈ 250ms   // 短促元素(如選項)
enter.base    ≈ 300ms   // 一般區塊淡入(0-4 §10 中段)
enter.slow    ≈ 350ms   // 重要句 / 結尾(0-4 §10 上限)
exit.base     ≈ 200~250ms // 題目淡出
```

**Easing(緩動)**

```
ease-out（柔和收尾）;不得使用彈跳 / 過衝 / 花俏曲線。
```

**Transform / Opacity**

```
opacity: 0 → 1
translateY: 10px → 0     // 唯一位移方向(承 0-4 §10)
不使用 X 軸滑動、縮放彈跳。
```

**Pause / Stagger / Hold(停頓、依序、停留)**

```
breath.short  ≈ 140~200ms   // 區塊之間的呼吸
breath.long   ≈ 400~600ms   // 重要時刻前的停頓(如結果開始、結尾前)
hold.insight  ≈ 1000~2000ms // 情緒高點句後的停留(Reading Pause);此時不觸發下一段
```

**Micro-interaction(微互動)**

```
hover.lift    ≈ translateY(-2px),300ms（承 0-4：選項輕微上浮）
select        邊框轉為鼠尾草綠,300ms;不閃爍、不放大。
```

**Reduced Motion**

```
尊重系統「減少動態」設定:關閉所有位移與淡入,直接呈現靜態內容。
```

# 9. Reading Time Goal｜閱讀時間目標

- 本產品**不追求快速讀完**。節奏設計要讓人願意慢慢讀。
- 題目頁:不催促;轉場永遠不早於內容被讀完;由使用者的節奏決定停留多久。
- 結果頁:保持**慢閱讀**,但避免過長——方向性指標約 **30~90 秒**。
  - 理由:一般使用者閱讀超過 90 秒可能開始疲勞;此為方向,不是需要達成的硬指標。
- **30~90 秒僅為方向性指標,不得以程式強制使用者停留**(不設強制等待、不以倒數解鎖按鈕等手段留人)。
- **不得鎖定捲動、阻擋使用者主動閱讀,亦不得使用任何形式的 scroll hijacking。**
- **禁止:** 自動跳頁、倒數計時、進度催促、任何「快點看完」的暗示。
- 一句話總結:寧可讓使用者**多停一會兒**,也不要讓他**快速滑過**;但也不要把節奏拖到讓人疲勞。
